# 设置菜单修复说明

## ✅ 问题已修复

### 问题原因
"设置"被配置为**顶级菜单项**，在 macOS 上不会正确显示。macOS 的菜单栏规范要求"设置"应该在应用名称的子菜单中。

### 修复内容
将"设置"移到了正确的位置：`mdParse` → `设置...`

**修复前**（错误）：
```
菜单栏: [mdParse] [文件] [编辑] [视图] [设置] [帮助] [窗口]
                                    ↑ 独立的顶级菜单（错误）
```

**修复后**（正确）：
```
菜单栏: [mdParse] [文件] [编辑] [视图] [帮助] [窗口]
         ↓
    [关于]
    [--------]
    [设置...] ← 在这里（正确）
    [--------]
    [退出]
```

## 🔍 如果还是不行

### 检查 1: 菜单是否更新
如果重启后菜单没有变化，可能需要：
```bash
# 完全退出应用
# 清除 Electron 缓存
rm -rf ~/Library/Application\ Support/mdparse
# 重新启动
npm run electron:dev
```

### 检查 2: 控制台输出
打开开发者工具（Cmd+Option+I），点击设置菜单，应该看到：
```
Settings menu clicked - opening settings modal
Rendering settings modal, showSettingsModal: true
```

### 检查 3: 手动测试
在控制台输入：
```javascript
// 手动触发设置界面
window.electronAPI.onMenuOpenSettings(() => {
  console.log('Manual trigger');
})();
```
